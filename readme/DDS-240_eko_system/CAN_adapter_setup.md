# Полное руководство по настройке CAN-адаптера CANable (candleLight) в Linux Mint

Этот документ содержит исчерпывающую информацию по настройке, диагностике и эксплуатации CAN-адаптера для связи с материнской платой на базе STM32H723.

---

## 1. Физическое подключение (Hardware Layer)

Для работы на высоких скоростях (1 Мбит/с) крайне важно соблюдать схему соединений и электрические параметры шины.

### Схема распиновки (Pinout)
| Адаптер CANable | Направление | Плата STM32H723 | Описание |
|:---:|:---:|:---:|:---|
| **CAN_H** | <---> | **CAN_H** | Линия данных High |
| **CAN_L** | <---> | **CAN_L** | Линия данных Low |
| **GND** | <---> | **GND** | **КРИТИЧЕСКИ ВАЖНО!** Общая земля для выравнивания потенциалов. Без неё возможны ошибки `ERROR-PASSIVE`. |

### Согласование линии (Termination)
*   **Резистор 120 Ом:** На адаптере CANable **обязательно** должен быть установлен джампер (перемычка) с надписью **120**. 
*   **Зачем:** На скорости 1 Мбит/с без терминатора сигнал отражается от концов проводов, что приводит к искажению данных и состоянию `BUS-OFF`.

## 2. Диагностика и драйверы (System Layer)

### История с драйвером slcand (SLCAN)
Первоначальная попытка запуска через последовательный порт:
`sudo slcand -o -c -s6 /dev/ttyACM0 can0`
**Ошибка:** `/dev/ttyACM0: No such file or directory`.
**Причина:** Адаптер прошит прошивкой **candleLight** (ID `1d50:606f`). Она не создает виртуальный COM-порт, а работает как нативное сетевое CAN-устройство.

### Проверка устройства в системе
Выполните команду:
```bash
lsusb
```
**Ожидаемый результат:** 
`Bus 003 Device 018: ID 1d50:606f OpenMoko, Inc. Geschwister Schneider CAN adapter`
*   **ID 1d50:606f** — это признак нативного SocketCAN устройства (драйвер `gs_usb`).

### Проверка логов ядра
```bash
dmesg | grep -i can
```
Должна появиться строка вида: `gs_usb 3-3.2:1.0 can0: low speed can device registered`.

## 3. Программная настройка (SocketCAN)

### Установка инструментов (can-utils)
```bash
sudo apt update
sudo apt install can-utils
```

### Тестирование в режиме Loopback (Петля)
Позволяет проверить адаптер "самого на себя" без внешних проводов.
```bash
# Остановить интерфейс
sudo ip link set can0 down

# Настроить 1Мбит/с и режим петли
sudo ip link set can0 type can bitrate 1000000 loopback on

# Поднять интерфейс
sudo ip link set can0 up
```
**Проверка:**
1. Откройте терминал: `candump can0`
2. Во втором терминале: `cansend can0 123#DEADBEEF`
3. Если в первом терминале появилось `can0 123 [4] DE AD BE EF` — адаптер и система исправны.

## 4. Настройка для проекта (1 Мбит/с)

Согласно файлу конфигурации `STM32H723_mother_board.ioc`, плата работает на скорости: **1 000 000 бит/с (1 Мбит/с)**.

### Команды для рабочего режима (с платой):
```bash
# 1. Остановить интерфейс
sudo ip link set can0 down

# 2. Настроить скорость и автоперезапуск (restart-ms 100)
sudo ip link set can0 type can bitrate 1000000 restart-ms 100 loopback off

# 3. Поднять интерфейс
sudo ip link set can0 up
```

## 5. Мониторинг и диагностика (Cheat Sheet)

### Просмотр состояния и ошибок
```bash
ip -details link show can0
```
*   **ERROR-ACTIVE:** Всё хорошо.
*   **ERROR-PASSIVE:** Проблемы с физикой (проверьте GND и джампер 120 Ом).
*   **BUS-OFF:** Критический сбой (автоматически сбросится через 100мс благодаря `restart-ms`).

### Команды для работы:
*   `candump can0` — простое прослушивание шины.
*   `candump -c -t d can0` — цветной вывод с метками времени.
*   `cansend can0 001#112233` — отправить тестовую команду на плату.
*   `ip -details -statistics link show can0` — статистика принятых/битых пакетов.

## 6. Автоматизация (Persistence)

Так как настройки SocketCAN сбрасываются при отключении USB, используйте один из методов ниже.

### Скрипт быстрого запуска (`init_can.sh`)
Создайте файл в корне проекта:
```bash
#!/bin/bash
sudo ip link set can0 down 2>/dev/null
sudo ip link set can0 type can bitrate 1000000 restart-ms 100 loopback off
sudo ip link set can0 up
echo "--- CAN0 Status ---"
ip -details link show can0 | grep -E "state|bitrate"
```
Сделайте его исполняемым: `chmod +x init_can.sh`.

### Автоматизация через Udev (Рекомендуется)
Чтобы адаптер сам поднимался при включении в USB:
1. Создайте файл: `sudo nano /etc/udev/rules.d/80-canable.rules`
2. Вставьте строку:
   `ACTION=="add", SUBSYSTEM=="net", ENV{ID_NET_DRIVER}=="gs_usb", RUN+="/sbin/ip link set %k type can bitrate 1000000 restart-ms 100", RUN+="/sbin/ip link set %k up"`
3. Примените правила:
   `sudo udevadm control --reload-rules && sudo udevadm trigger`

## 7. Основы автоматизации: Работа с параметрами Bash

Чтобы сделать скрипты гибкими, используются **позиционные переменные** (`$1`, `$2` и т.д.).

### Как это работает в `init_can.sh`:
```bash
# ${1:-1000000} означает:
# Взять первый параметр после имени скрипта ($1).
# Если параметр не передан, использовать значение 1000000 по умолчанию.
BITRATE=${1:-1000000}
```

### Примеры запуска:
1. `./init_can.sh` — `$1` пуст, установится скорость **1 000 000**.
2. `./init_can.sh 500000` — `$1` станет равным **500 000**, установится эта скорость.

### Как добавить второй параметр:
Если вам нужно передать второе значение (например, переменную `X` с дефолтом 20):
```bash
X=${2:-20}
```
*Запуск:* `./init_can.sh 1000000 50` — здесь `$1=1000000`, `$2=50`.

### Полезные переменные Bash:
* `$0` — имя самого скрипта.
* `$1, $2...` — первый, второй и т.д. параметры.
* `$#` — общее количество переданных параметров.
* `fi` — обязательное завершение блока `if` (слово `if` наоборот).
