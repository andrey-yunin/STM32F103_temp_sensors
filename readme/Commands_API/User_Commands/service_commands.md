# Справочник сервисных команд DDS-240 (Service Layer)

Данный документ описывает расширенный набор команд в инженерном диапазоне (0xFxxx), предназначенный исключительно для сервисного обслуживания, пусконаладки и инсталляции оборудования. Эти команды не используются в основном приложении пользователя.

## 1. Универсальные сервисные команды (0xF0xx)

Команды, поддерживаемые всеми исполнительными модулями системы.

### 0xF001 - SRV_GET_DEVICE_INFO
Запрос расширенной информации об устройстве для идентификации его типа и уникального экземпляра.

**Параметры:** нет

**Ответ (DATA) — последовательность из 3-х фреймов:**

#### Фрейм 1 (Metadata & ID Start)
| Байт | Поле | Описание |
|------|------|----------|
| 0 | type_id | Тип устройства (0x40 - Thermo, 0x20 - Motion) |
| 1 | fw_major | Версия прошивки (Major) |
| 2 | fw_minor | Версия прошивки (Minor) |
| 3 | hw_config | Конфигурация (например, кол-во каналов: 8) |
| 4-5 | uid[0..1] | Начало 96-битного UID чипа |

#### Фрейм 2 (ID Middle)
| Байт | Поле | Описание |
|------|------|----------|
| 0-5 | uid[2..7] | Средняя часть 96-битного UID чипа |

#### Фрейм 3 (ID End)
| Байт | Поле | Описание |
|------|------|----------|
| 0-3 | uid[8..11] | Финальная часть 96-битного UID чипа |
| 4-5 | reserved | Зарезервировано (0x00) |

---

---

### 0xF002 - SRV_REBOOT
Дистанционная перезагрузка модуля.

**Параметры:** 
| Поле | Тип | Описание |
|------|-----|----------|
| magic | UINT16 | Защитный ключ (всегда 0xDEAD) |

**Ответ:** ACK, затем устройство перезагружается.

---

### 0xF003 - SRV_FLASH_COMMIT
Принудительное сохранение текущих настроек из RAM в энергонезависимую память (Flash).

**Параметры:** нет
**Ответ:** ACK и DONE (после завершения записи)

---

### 0xF004 - SRV_FACTORY_RESET
Сброс всех настроек модуля к заводским значениям и очистка Flash-памяти.

**Параметры:**
| Поле | Тип | Описание |
|------|-----|----------|
| magic | UINT16 | Защитный ключ (всегда 0xFACE) |

**Ответ:** ACK и DONE

---

## 2. Сервисные команды термодатчиков (0xF1xx)

Команды для работы с шиной 1-Wire и конфигурирования логических каналов.

### 0xF101 - SRV_SCAN_1WIRE
Запуск принудительного поиска всех физических устройств на шине 1-Wire.

**Параметры:** нет

**Ответ (DATA):**

| Поле | Тип | Описание |
|------|-----|----------|
| found_count | UINT8 | Количество обнаруженных физических ID |

---

### 0xF102 - SRV_GET_PHYS_ID
Запрос 64-битного уникального ID (ROM-кода) физического датчика по его индексу в списке обнаружения.

**Параметры:**
| Поле | Тип | Описание |
|------|-----|----------|
| phys_idx | UINT8 | Индекс в списке (0..found_count-1) |

**Ответ (DATA):**

| Поле | Тип | Описание |
|------|-----|----------|
| phys_idx | UINT8 | Подтверждение индекса |
| rom_id | UINT8[8] | 64-битный уникальный адрес датчика |

---

### 0xF103 - SRV_SET_CHANNEL_MAP
Привязка физического ROM ID к логическому номеру канала ("Реагент 1", "Реагент 2" и т.д.).

**Параметры:**
| Поле | Тип | Описание |
|------|-----|----------|
| logical_id | UINT8 | Логический номер (0..7) |
| rom_id | UINT8[8] | 64-битный ROM-код датчика |

**Ответ:** ACK и DONE. 
**Примечание:** Изменения вступают в силу немедленно в RAM, но требуют `SRV_FLASH_COMMIT` для сохранения после перезагрузки.

---

### 0xF104 - SRV_GET_CHANNEL_MAP
Запрос текущей привязки физического ID для указанного логического канала.

**Параметры:**
| Поле | Тип | Описание |
|------|-----|----------|
| logical_id | UINT8 | Логический номер (0..7) |

**Ответ (DATA):**

| Поле | Тип | Описание |
|------|-----|----------|
| logical_id | UINT8 | Подтверждение логического номера |
| rom_id | UINT8[8] | Текущий привязанный ROM ID (все 0xFF если не назначен) |

---

## 3. Процедура инсталляции (Mapping)

Для настройки системы "ПО Инсталятора" должно выполнить следующие шаги:

1. Отправить `SRV_SCAN_1WIRE` для получения количества датчиков.
2. Циклически вызывать `SRV_GET_PHYS_ID` для получения всех доступных адресов.
3. Отобразить пользователю текущие температуры для каждого физического ID (используя стандартную команду `SENSOR_GET_TEMP` по индексу).
4. После идентификации (нагрева датчика) отправить `SRV_SET_CHANNEL_MAP` для привязки ID к нужному логическому имени.
5. Отправить `SRV_FLASH_COMMIT` для фиксации настроек в памяти платы.
